# Contributing to Behind PRISMA's Success

Thank you for your interest in contributing to this research project! This document provides guidelines for contributing to the repository.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if the issue already exists in the [issue tracker](https://github.com/mq3n/Behind-PRISMA-s-Success/issues)
2. If not, create a new issue with a clear title and description
3. Include relevant details such as:
   - Steps to reproduce (for bugs)
   - Expected vs. actual behavior
   - Your environment (OS, Python version, etc.)

### Submitting Changes

1. **Fork the repository** to your own GitHub account
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Behind-PRISMA-s-Success.git
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** following the coding standards below
5. **Test your changes** thoroughly
6. **Commit your changes** with clear, descriptive messages:
   ```bash
   git commit -m "Add feature: description of what you added"
   ```
7. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Submit a Pull Request** from your fork to the main repository

## Coding Standards

- Follow PEP 8 style guide for Python code
- Write clear, descriptive variable and function names
- Add comments for complex logic
- Include docstrings for functions and classes
- Keep functions focused and modular

## Documentation

- Update the README.md if you add new features
- Document new functions and classes with docstrings
- Add comments to explain non-obvious code

## Testing

- Add tests for new functionality
- Ensure all existing tests pass before submitting a PR
- Run tests locally with: `pytest`

## Code Review Process

1. Maintainers will review your Pull Request
2. They may request changes or ask questions
3. Once approved, your PR will be merged

## Questions?

If you have questions, feel free to open an issue or reach out to the maintainers.
